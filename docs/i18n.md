# 国际化（i18n）指南

Argus 内置零依赖的 i18n 系统，支持 9 种语言，并可扩展任意语言

## 支持的语言

| 语言 | 标签 | 说明 |
|------|------|------|
| 英文 | `en` | 默认语言，也是回退语言 |
| 简体中文 | `zh` | |
| 繁体中文 | `zh-TW` | |
| 日语 | `ja` | |
| 韩语 | `ko` | |
| 法语 | `fr` | |
| 德语 | `de` | |
| 西班牙语 | `es` | |
| 俄语 | `ru` | |

---

## 统一 i18n 体系

Argus 的所有翻译数据统一由 `i18n` 子包管理，无论是字段级翻译（struct tag 校验错误）还是校验器内部消息（compare / format / json / network / schema 模块错误），都存储在同一个 `i18n.store` 中，通过同一套 API 访问。

| 消息类型 | key 示例 | 说明 |
|---------|---------|------|
| 字段级翻译 | `required` → `{field} is required` | struct tag 校验错误翻译 |
| 校验器内部消息 | `format.email_empty` → `email address is empty` | 各校验模块的错误消息 |

`validator` 根包和 `validate` 子包均通过 `i18n` 包获取翻译，不存在重复存储

## 快速开始

### 1. 设置全局语言

```go
import "github.com/kamalyes/go-argus"

// 设置为中文
validator.SetLocale("zh")

// 获取当前语言
locale := validator.GetLocale() // "zh"
```

### 2. 翻译 struct tag 校验错误

```go
type User struct {
    Name  string `json:"name" validate:"required,min=2"`
    Email string `json:"email" validate:"required,email"`
}

v := validator.New()
err := v.Struct(User{})

// 翻译为中文
messages := validator.TranslateValidationErrors(err, "zh")
// [{field: "name", message: "name 为必填字段", tag: "required"}, ...]

// 翻译为英文
messages := validator.TranslateValidationErrors(err, "en")
// [{field: "name", message: "name is required", tag: "required"}, ...]
```

### 3. 校验器内部消息自动跟随语言

```go
import "github.com/kamalyes/go-argus"

validator.SetLocale("zh")

result := validator.ValidateEmail("")
fmt.Println(result.Message) // Email 地址为空

validator.SetLocale("en")
result = validator.ValidateEmail("")
fmt.Println(result.Message) // email address is empty

validator.SetLocale("ja")
result = validator.ValidateEmail("")
fmt.Println(result.Message) // メールアドレスが空です
```

## API 参考

### 根包 API

| 函数 | 说明 |
|------|------|
| `validator.SetLocale(locale)` | 设置全局语言环境 |
| `validator.GetLocale()` | 获取当前语言环境 |
| `validator.RegisterI18n(locale, key, template)` | 注册单个翻译模板 |
| `validator.RegisterI18nMessages(locale, map)` | 批量注册翻译模板 |
| `validator.RegisterTranslation(locale, tag, template)` | 注册字段级翻译模板 |
| `validator.RegisterTranslations(locale, map)` | 批量注册字段级翻译模板 |
| `validator.TranslateValidationErrors(err, locale)` | 将校验错误翻译为数组化消息 |

### i18n 子包 API

| 函数 | 说明 |
|------|------|
| `i18n.SetLocale(locale)` | 设置全局语言环境 |
| `i18n.GetLocale()` | 获取当前语言环境 |
| `i18n.Msg(key, args)` | 查找并渲染消息模板 |
| `i18n.Lookup(locale, key)` | 查找指定语言的消息模板 |
| `i18n.Register(locale, key, template)` | 注册单个翻译模板 |
| `i18n.RegisterMessages(locale, map)` | 批量注册翻译模板 |
| `i18n.Normalize(locale)` | 规范化语言标签 |
| `i18n.EnMessages()` | 获取英文消息模板集合 |
| `i18n.ZhMessages()` | 获取简体中文消息模板集合 |
| `i18n.ZhTWMessages()` | 获取繁体中文消息模板集合 |
| `i18n.JaMessages()` | 获取日语消息模板集合 |
| `i18n.KoMessages()` | 获取韩语消息模板集合 |
| `i18n.FrMessages()` | 获取法语消息模板集合 |
| `i18n.DeMessages()` | 获取德语消息模板集合 |
| `i18n.EsMessages()` | 获取西班牙语消息模板集合 |
| `i18n.RuMessages()` | 获取俄语消息模板集合 |

## 回退机制

当指定语言没有对应翻译时，自动回退到英文（`en`）如果英文也没有，返回 key 本身

```
当前语言 → en → key
```

## 自定义翻译

### 注册新的语言

```go
validator.RegisterI18nMessages("pt", map[string]string{
    "format.email_empty": "o endereço de e-mail está vazio",
    "format.email_invalid": "formato de e-mail inválido: {error}",
})
validator.SetLocale("pt")
```

### 覆盖已有翻译

```go
validator.RegisterI18n("zh", "required", "{field} 不能为空哦")
```

### 注册字段级翻译模板

字段级翻译支持以下占位符：

| 占位符 | 说明 |
|--------|------|
| `{field}` | 字段展示名（优先使用 json tag） |
| `{namespace}` | 字段路径 |
| `{tag}` | 规则名 |
| `{param}` | 规则参数 |
| `{value}` | 字段值 |

```go
validator.RegisterTranslation("zh", "custom_rule", "{field} 不满足自定义规则 {param}")
```

## ValidationMessage 结构

`TranslateValidationErrors` 返回的每条消息结构：

```go
type ValidationMessage struct {
    Field           string      `json:"field"`                      // 字段展示名
    Namespace       string      `json:"namespace"`                  // 字段路径
    StructField     string      `json:"struct_field,omitempty"`     // 结构体字段名
    StructNamespace string      `json:"struct_namespace,omitempty"` // 结构体字段路径
    Tag             string      `json:"tag"`                        // 规则名
    ActualTag       string      `json:"actual_tag"`                 // 实际规则名
    Param           string      `json:"param,omitempty"`            // 规则参数
    Value           interface{} `json:"value,omitempty"`            // 字段值
    Message         string      `json:"message"`                    // 翻译后的消息
}
```

### JSON 输出示例

```json
[
  {
    "field": "email",
    "namespace": "User.email",
    "tag": "required",
    "message": "email 为必填字段"
  },
  {
    "field": "age",
    "namespace": "User.age",
    "tag": "gte",
    "param": "0",
    "value": -1,
    "message": "age 必须大于或等于 0"
  }
]
```

## 常量管理

所有校验器内部消息 key 统一在 `validate/constants.go` 中以常量形式维护：

```go
// format 系列
const MsgFormatEmailEmpty    = "format.email_empty"
const MsgFormatEmailInvalid  = "format.email_invalid"
const MsgFormatIPInvalid     = "format.ip_invalid"

// compare 系列
const MsgCompareNumberFailed = "compare.number_failed"

// schema 系列
const MsgSchemaTypeMismatch  = "schema.type_mismatch"
```

新增消息 key 时，请先在 `constants.go` 中定义常量，再在 `i18n` 子包的对应语言文件（如 `en.go`、`zh.go`）中添加翻译模板
